What is the Document Object Model (DOM)?

The HTML DOM is a standard object model and programming interface for HTML. It defines:

  • - The HTML elements as objects.
  • - The properties of all HTML elements.
  • - The methods to access all HTML elements.
  • - The events for all HTML elements.

In other words: The HTML DOM is a standard for how to get, change, add, or delete HTML elements.

How do you select an element from the DOM?

  • - By ID: Select an element using its unique ID
  • - By class: Select multiple elements with the same class name.
  • - By tag name: Select all elements of a specific type.
  • - By query selector: Use CSS selectors to select elements based on various criteria, such as ID, class, tag name, attributes, and more

What is event delegation in the context of the DOM, and why is it useful?

  • - Improved performance: When dealing with a large number of child elements, event delegation can significantly improve performance, as it avoids creating and managing multiple event listeners.
  • - Dynamic content: If the number of child elements changes dynamically (e.g., through JavaScript or AJAX), event delegation ensures that new elements will automatically inherit the event listener from their parent.
  • - Simplified code: Event delegation can lead to cleaner and more concise code, as you only need to manage a single event listener for a group of related elements.

How do you manipulate an element's attributes and styles using the DOM?

  • - Get an attribute using a method that retrieves it by name.
  • - Set an attribute by specifying the attribute name and value.
  • - Remove an attribute using a method that takes the attribute name as a parameter.
